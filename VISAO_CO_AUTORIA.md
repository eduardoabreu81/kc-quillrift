## 1. DECLARAÇÃO DE VISÃO

> KC-QuillRift não é um editor com IA.  
> É um **co-autor inteligente** que constrói ficção junto com você.  
> Em cada seção, em cada decisão, a IA está presente — sugerindo, criando, revisando, conectando.

### Nossa Ambição

Ser o **melhor studio de co-autoria IA** já criado, combinando:
- ✅ A experiência visual do **Sudowrite**
- ✅ O controle de contexto do **NovelCrafter**
- ✅ A simplicidade open source do **Writingway**
- ✅ A sofisticação técnica do **SAGA**
- ✅ E no futuro: nossa **própria LLM especializada** em ficção

## 2. ARQUITETURA DE CO-AUTORIA

### 2.1 Não é uma ferramenta. É um parceiro.

**Editor tradicional:** Você escreve, o editor formata.  
**KC-QuillRift:** Você pensa, o co-autor propõe, você decide.

```
┌─────────────────────────────────────────────────────────────────┐
│                    CO-AUTOR ORQUESTRADOR                        │
│                                                                 │
│  Entende intenção → Seleciona agentes → Coordena criação       │
│  ←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←  │
│         Apresenta opções → Aguarda decisão do autor             │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 Agentes Especialistas (Squad de Co-Autores)

Cada agente é um **especialista** com memória, estilo e objetivo próprio.

| Agente | Função | Inspiração | Personalidade |
|--------|--------|------------|---------------|
| **🏛️ Architect** | Estrutura narrativa, outline, beats | Sudowrite Story Bible | Visionário, estruturado |
| **📚 Librarian** | Worldbuilding, conexões, lore | SAGA Knowledge Graph | Curioso, meticuloso |
| **🎭 VoiceSmith** | Diálogos, voz de personagem | Sudowrite Write | Dramaturgo, sensível |
| **✍️ Builder** | Escrita de cenas, prosa | Writingway | Versátil, adaptável |
| **🔍 RedPen** | Revisão, consistência, qualidade | SAGA Validation | Crítico, preciso |
| **⏳ Historian** | Timeline, cronologia, eventos | SAGA Graph | Analítico, visual |
| **🎨 Describer** | Cenários sensoriais (5 sentidos) | Sudowrite Describe | Poético, sensorial |
| **💡 Ideator** | Brainstorming, nomes, títulos | Sudowrite Brainstorm | Criativo, abundante |
| **🧩 ConsistencyKeeper** | Validação de lore, contradições | NovelCrafter + SAGA | Vigilante, detalhista |

### 2.3 Modos de Co-Autoria

O autor escolhe como quer trabalhar:

| Modo | Descrição | Quando usar |
|------|-----------|-------------|
| **🤝 Colaborativo** | IA sugere enquanto você escreve (autocomplete inteligente) | Flow state, escrita rápida |
| **💭 Consultivo** | Você pede, IA propõe 3 opções, você escolhe | Decisões estratégicas |
| **⚡ Gerador** | Você descreve, IA cria completo | Superar bloqueio, rascunho rápido |
| **🔎 Analítico** | IA lê e aponta problemas | Revisão, polimento |
| **🎓 Tutor** | IA explica técnicas, sugere leituras | Aprender, evoluir |

## 4. ARQUITETURA TÉCNICA PARA CO-AUTORIA

### 4.1 Sistema de Agentes

```typescript
interface Agent {
  id: string;
  name: string;
  role: AgentRole;
  personality: string; // System prompt específico
  skills: Skill[];
  memory: AgentMemory; // Conversas anteriores com este autor
  active: boolean;
}

interface AgentOrchestrator {
  // Decide qual(is) agente(s) chamar baseado na intenção
  selectAgents(intent: UserIntent): Agent[];
  
  // Coordena múltiplos agentes simultâneos
  coordinate(agents: Agent[], context: Context): AgentOutput[];
  
  // Sintetiza outputs de múltiplos agentes
  synthesize(outputs: AgentOutput[]): Suggestion[];
}
```

### 4.2 Contexto Rico para Cada Agente

```typescript
interface AgentContext {
  // O que este agente precisa saber
  project: ProjectSummary;      // Resumo do projeto (todos)
  bible: BibleSubset;           // Parte relevante da bíblia
  timeline: TimelineWindow;     // Eventos próximos
  characters: Character[];      // Personagens na cena
  previousScenes: Scene[];      // Contexto imediato
  authorStyle: StyleProfile;    // Aprendido do autor
  
  // Memória do agente
  previousInteractions: Interaction[];
  authorPreferences: Preference[];
  avoidedPatterns: string[];    // O que o autor não gosta
}
```

### 4.3 Pipeline de Geração

```
Intenção do Autor
       │
       ▼
┌──────────────────┐
│ Orchestrator     │ ← Entende o que o autor quer
└────────┬─────────┘
         │
         ▼

         │
         ▼

         ▼

```

## 5. FUTURO: NOSSA PRÓPRIA LLM

### 5.1 Por que treinar uma LLM especializada?

| Problema | Solução |
|----------|---------|
| GPT-4 é caro e genérico | Nossa LLM: otimizada para ficção, custo baixo |
| Claude não entende arcos narrativos | Nossa LLM: treinada em story structure |
| Modelos genéricos "esquecem" personagens | Nossa LLM: memória nativa de lore |
| Dependência de terceiros | Nossa LLM: soberania tecnológica |

### 5.2 Roadmap para LLM Própria

#### Fase 1: Foundation (Ano 1)
- Coletar dataset de ficção de qualidade (domínio público + licenciado)
- Fine-tune de modelo open source (Llama, Mistral, DeepSeek)
- Especialização em:
  - Continuação de cenas
  - Diálogos in-character
  - Descrições sensoriais

#### Fase 2: Especialização (Ano 2)
- Treinamento com RLHF de autores reais
- Integração nativa com Knowledge Graph
- Memória de longo prazo (100k+ tokens)
- Geração estruturada (JSON para entidades, YAML para outline)

#### Fase 3: Autonomia (Ano 3)
- Modelo próprio (não mais fine-tune)
- Arquitetura otimizada para ficção
- Custo 10x menor que GPT-4
- Performance superior em benchmarks de ficção
